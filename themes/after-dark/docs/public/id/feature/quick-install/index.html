Instal Cepat

Hanya satu perintah yang anda butuhkan untuk membuat website baru.

After Dark termasuk skrip instalasi portabel untuk pengaturan cepat. Silahkan instal Hugo 0.51 atau yang lebih besar sebelum menjalankan skrip:

 22vers() { echo "$@" | awk -F. '{ printf("%d%03d%03d%03d\n", $1,$2,$3,$4); }'; }
 23parse_hugo_vers() {
 24   hugovers=$(hugo version | sed 's/-.*//')
 25   hugovers="${hugovers#"${hugovers%%[0-9]*}"}"
 26   HUGO_VERSION=${hugovers}
 27}
 28
 29validate_hugo() {
 30   # Exit with error if hugo is not installed
 31   if ! hash hugo 2>/dev/null; then
 32      echo "Error: After Dark requires Hugo version 0.51 or greater" >&2
 33      exit 1
 34   fi
 35
 36   # Exit with error if not minimum required hugo version
 37   parse_hugo_vers
 38   if [ "$(vers "$HUGO_VERSION")" -lt "$(vers "0.51")" ]; then
 39      echo "Error: After Dark requires Hugo version 0.51 or greater"
 40      exit 1
 41   fi
 42}

 44confirm_intent() {
 45   while true; do
 46      printf "Do you wish to install After Dark? (y/n) "
 47      read -r yn </dev/tty
 48      case $yn in
 49      [Yy]*)
 50         break
 51         ;;
 52      [Nn]*) exit ;;
 53      *) echo "Please answer yes or no." ;;
 54      esac
 55   done
 56}

 62prompt_site_name() {
 63   printf "Please enter a name for your site: "
 64   read -r response </dev/tty
 65   SITE_NAME="$(echo "$response" | tr -cd '[:alnum:] [:space:]' | tr -s '[:space:]')"
 66   while true; do
 67      printf "You entered %s. Is that correct? (y/n) " "\"$SITE_NAME\""
 68      read -r yn </dev/tty
 69      case $yn in
 70      [Yy]*)
 71         break
 72         ;;
 73      [Nn]*)
 74         prompt_site_name
 75         break
 76         ;;
 77      *) echo "Please answer yes or no." ;;
 78      esac
 79   done
 80}

 82format_dir_name() {
 83   echo "$1" | tr '[:upper:]' '[:lower:]' | tr ' ' '-' | tr -cd '[:alnum:]._-'
 84}
 85
 86set_default_install_dir() {
 87   SITE_DIR="$(format_dir_name "$SITE_NAME")"
 88}
 89
 90prompt_install_dir() {
 91   default_dir="$(format_dir_name "$SITE_NAME")"
 92   printf "Choose an installation directory name (%s): " "$default_dir"
 93   read -r response </dev/tty
 94   if [ "$response" = "" ]; then
 95      SITE_DIR=$default_dir
 96   elif [ "${response%"${response#?}"}" = "/" ]; then
 97      echo "Directory name not allowed. Proceeding with default..."
 98      SITE_DIR=$default_dir
 99   else
100      formatted_input_dir=$(format_dir_name "$response")
101      if [ "$formatted_input_dir" = "" ]; then
102         echo "Directory name not allowed. Proceeding with default..."
103         SITE_DIR=$default_dir
104      else
105         SITE_DIR=$formatted_input_dir
106      fi
107   fi
108   while true; do
109      printf "Installation directory name set to %s. Is that correct? (y/n) " "\"$SITE_DIR\""
110      read -r yn </dev/tty
111      case $yn in
112      [Yy]*)
113         break
114         ;;
115      [Nn]*)
116         prompt_install_dir
117         break
118         ;;
119      *) echo "Please answer yes or no." ;;
120      esac
121   done
122}

135download_theme() {
136   echo "Downloading the latest version of After Dark ..."
137   LATEST_META=$(wget -qO- https://registry.npmjs.org/after-dark/latest)
138   vers=$(echo "$LATEST_META" | grep -oE "\"version\".*[^,]*," | cut -d ',' -f1 | cut -d ':' -f2 | tr -d '" ')
139   mkdir -p themes/after-dark
140   wget -qO- https://registry.npmjs.org/after-dark/-/after-dark-"$vers".tgz | tar --strip-components=1 -xz -C themes/after-dark
141   echo "Version $vers downloaded to $SITE_DIR/themes/after-dark"
142}
143
144download_module() {
145   [ -z "$1" ] && {
146      echo "Error: Attempt to download undefined module" >&2
147      exit 1
148   }
149   echo "Downloading $1 module for After Dark ..."
150   meta=$(wget -qO- https://registry.npmjs.org/"$1"/latest)
151   vers=$(echo "$meta" | grep -oE "\"version\".*[^,]*," | cut -d ',' -f1 | cut -d ':' -f2 | tr -d '" ')
152   mkdir -p themes/"$1"
153   wget -qO- https://registry.npmjs.org/"$1"/-/"$1"-"$vers".tgz | tar --strip-components=1 -xz -C themes/"$1"
154   echo "Version $vers downloaded to $SITE_DIR/themes/$1"
155}

Skrip telah diuji pada GNU/Linux dan BSD (Darwin) dan Windows melalui Cmder.

Warning: Periksa skrip yang diunduh dari Internet sebelum menjalankannya.

Berikut adalah tiga metode untuk men-download dan menjalankan:

  1. Unduh dan kirim ke sh secara langsung:

    wget -qO- https://codeberg.org/vhs/after-dark/raw/branch/trunk/bin/install | sh

  2. Unduh di file baru, chmod dan jalankan:

    curl -O https://cdn.jsdelivr.net/npm/after-dark@latest/bin/install && \
chmod +x install && ./install

  3. Dari kanonik git klon:

    # sumber klon dan ubah ke direktori sumber git klon https://codeberg.org/vhs/after-dark.git && cd "$_"

# gunakan npm cli untuk mendapatkan hash rilis
echo "${$(npm run integrity)#*sha512-}"

# jalankan instal cepat setelah memvalidasi
./bin/install

Skrip harus selesai dalam 5-10 detik sehingga menghasilkan sampel situs dan dokumen bantuan:

Situs baru Anda akan disebut flying-toaster. Ubah ke nama proyek anda kapan saja anda suka. Akses situs dengan menavigasi ke https://localhost:1313.

Konfigurasi Multi-situs (Lanjutan)

After Dark memungkingkan menjalankan manajemen multi situs dari satu instalasi. Untuk mengelola beberapa website gunakan -c dan -d bendera untuk menentukan konten dan tujuan direktori secara berurutan.

Misalnya, untuk mengahasilkan situs audio menggunakan instalasi After Dark saat ini, buat skrip yang dapat dieksekusi untuk menghasilkan situs tersebut:

flying-toasters/bin/gen-audio-site
#!/bin/sh
hugo -c sites/audio -d public/static.domain.example

Dimana audio berisi konten untuk situs itu:

 layouts
 sites
  audio
      audiobooks
           gaining-currency.md
         the-power-of-now.md
      clips
          war-of-the-worlds.md
 static

Dan public berisi folder untuk setiap situs:

public
 static.domain.example
     categories
      index.xml
     audiobooks
        index.html
     clips
      index.html
     css
     index.html
     index.xml
     js
     sitemap.xml
     tags
         index.xml

Dan buat skrip lain untuk menyajikan konten untuk diedit:

flying-toasters/bin/serve-audio-site
#!/bin/sh
hugo -c sites/audio

Setiap subdirektori public kemudian menjadi website yang independen dan dapat disebarkan serta salinan persisnya disimpan untuk konten tujuan yang dihasilkan.

Tip: Untuk jenis fleksibiltas tambahan hugo --help dan modifikasi skrip anda menggunakan --theme dan --config flags.

Multi situs sangat cocok untuk mempertahankan tampilan dan rasa yang konsisten di berbagai domain asal sekaligus membatasi kebutuhan untuk menjalankannya Upgrade Script untuk setiap situs.
